== 커널 ==
>> 공통 <<
- Google 의 8월 보안 패치들 적용
- Google 의 Android O Preview 4 패치들 적용
- sdcardfs: Google android-3.10 으로 부터 최신 패치들 적용
- f2fs: f2fs-stable-4.13-rc2-4.9 업스트림 패치들 병합
- power: bcl: 불필요한 __ref 와 __refdata 플래그들 제거
- msm_thermal: 'temp_little_off_threshold' 증가 (균형 있는 성능 되돌림.)

- clk: qcom: clock-pll: myfluxi 님이 패치하신 수정 패치들 적용
- scsi: sg, af_key, perf/core: Google android-3.10 으로 부터 일부 업스트림 패치들 적용
- arm64: PAGE_OFFSET 에 있는 오버 플로우 방지 (Linux master 업스트림)
- s2w, dt2w: 진동 on/off 옵션 추가
- ext4: Linux master 업스트림으로 부터 일부 오버플로우와 에러 수정 패치들 적용
- qcacld-2.0(wlan 드라이버): LA.BF64.1.2.3_rb1.15 로 업데이트

- multirom: trampoline: /proc/cmdline 출력 완전히 다룰수 있도록 함
- Updater: 라이브러리들 업데이트
- Updater: 다이제스트를 md5 에서 sha256 으로 변경
- Updater: v1.10 으로 버젼 업
- MultiROMMgr: 라이브러리들 업데이트
- MultiROMMgr: 다이제스트를 md5 에서 sha256 으로 변경
- aroma: s2w, dt2w: 진동 on/off 옵션 추가

<b><#selectbg_g>[bullhead_II-v25.0]</#></b>
== 커널 ==
>> 공통 <<
- Google 의 7월 패치들과 Android O DP3 패치들 적용
- Linux 3.10.107
- xpad: 최신 Linux 업스트림 패치들 적용
- kms: 최신 Linux 업스트림 패치들 적용
- net, mm, sdcardfs: Google android-3.10 로 부터 최신 패치들 적용
- msm_thermal: 낮은 온도에서 더 부드러운 성능을 낼 수 있도록 조정!
- msm_thermal: 쓰로틀링 지연 구현
(주파수가 바로 제한되지 않고, 약간의 지연이 된 후에 제한이 됩니다.
 또한, 온도가 내려가면 제한을 해제하고, 지연시간을 초기화 시킵니다.
 따라서, 이 시간동안 성능을 보장합니다. 기본값은 4초입니다.)
- msm_thermal: Little 의 코어 컨트롤 구현
(성능과 온도가 일부 Little 코어들을 끄고, 남아있는 코어들에게
 높은 주파수들을 주는 것이 더 좋습니다.)
- qcacld-2.0(wlan driver): LA.BF64.1.2.3_rb1.14 로 업데이트 (from CAF)
