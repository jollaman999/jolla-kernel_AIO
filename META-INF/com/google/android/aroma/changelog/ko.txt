>> N <<

>> O <<

>> 공통 <<
- arp: 'ignore_proxy_arp' 옵션을 추가하고 기본으로 활성화 함.
(모바일 장치에서 proxy ARP 를 사용할 필요가 없습니다.
또한, 이 패킷들을 무시하는 것이 안전합니다.)
- arp: Proxy ARP 에서 게이트웨이 업데이트 탐지 제거.
(Proxy ARP 에서 게이트웨이 업데이트를 탐지하는 것이 말이 안됩니다.
Proxy ARP 를 사용하면, 트래픽들을 포워드 시키기 위해 게이트웨이 IP 주소를 사용하지 않기 때문입니다.)
- arp: 게이트웨이로 향하는 ARP 요청들을 확인하여 공격자의 하드웨어 주소 감지.
- arp: arp_project: v1.0 으로 버젼업
- ANDROID: hid: uhid: implement refcount for open and close (from Google android-3.10)
- msm: kgsl: Add accounting for memory mapped in userspace
(누가의 mr2 패치들을 적용하는 동안 빠져 있었습니다.)
- CAF 의 보안 관련 패치들 일부 적용.

== 커널 ==
- Android O DP2 패치들 적용
- f2fs: 'f2fs_submit_discard_endio()' 에서 빠진 'bio_put(bio)' 추

- kernel: 데몬이 작동중일 때만 su 가 실행되도록 함 (from LineageOS android_kernel_oneplus_msm8996)
- kernel: su 체크에서 잠재적인 refcount 누수 수정  (from LineageOS android_kernel_oneplus_msm8996)
- sdcardfs: Google android-3.10 으로 부터 최신 android-3.10 패치들 적용
- net/packet: Google android-3.10 으로 부터 오버플로우 패치들 적용
- msm_thermal: 높은 온도를 체크하여 LITTLE 의 주파수를 낮은 값으로 감소시킴
( 빅코어가 꺼졌을 때, LITTLE 의 주파수 감소 스텝은 'temp_count_max_little_with_big_off'
에 의해 2가 됩니다.
 몇몇의 상황에서, 장치가 매우 뜨거울 경우, 주파수를 감소시킬 방법이 없습니다.
 LITTLE 코어들이 낮은 주파수를 얻기 위해 높은 온도를 체크할 필요가 있습니다.)
- arm/arm64: Google android-3.10 으로 부터 컴파일 관련 일부 패치들 적용
