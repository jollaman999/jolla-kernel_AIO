<b><#selectbg_g>[bullhead_II-v14.0]</#></b>
== 커널 ==
>> 공통 <<
- mdss, sovc: 균형이 맞지 않는 mdss 끄기 수정
(진짜 문제는 균형이 맞지 않는 mdss vreg(전원) 상태였습니다.
사운드가 정지되고 난 후 또 다시 끄려고 시도합니다.
따라서 균형이 맞지 않는 상태를 초래하게 되고 디바이스가 화면을 다시 켤 수 없게 됩니다.
이미 꺼져 있는 경우에 mdss vreg(전원)을 끄지 않도록 하였습니다.)
- mdss, sovc: 저전력 모드에 진입 했을시 mdss 를 끄지 않음
(mdss 에 저전력 모드가 있습니다.
이 모드는 디바이스가 알람을 받아서 화면을 검정과 흰색으로 켤때 사용됩니다.
따라서 저전력 모드로 들어갔을때 mdss 를 끄면 안됩니다.
그렇지 않으면 타임아웃 에러 메세지들을 발생시킵니다.)
- 변경사항 취소 "block: maple: 디스플레이 상태를 필요할 때 체크"
(필요없는 커밋.)
- Google android-3.10 로 부터 sdcardfs 와 관련된 Android mnt 변경사항 적용
- android: binder: Google 의 hwbinder 브랜치 변경 사항들 적용
- net: Google 의 hwbinder 브랜치 변경 사항들 적용
- sound: msm, soc: 사이즈 최적화
- lib/vsprintf.c: 업스트림 변경 사항들 적용
- Input: xpad: 업스트림 변경 사항들 적용
- ksm, msm_thermal, dt2w, s2w, sovc: fb notifier 에서 저전력 상태 체크
(mdss 에 저전력 모드가 있습니다.
이 모드는 디바이스가 알람을 받아서 화면을 검정과 흰색으로 켤때 사용됩니다.
만약 사용자가 자동 잠금 해제 기능을 활성화 하였다면, mdss 가 화면을 저전력 모드로 킬때 문제가 발생할 것입니다.
드라이버가 resume 상태로 전환되지 않습니다.
따라서 이 저전력 상태를 fb notifier 에서 체크하여야 합니다.)
